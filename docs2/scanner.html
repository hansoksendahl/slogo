Slogo is generated by the Jison javascript library
using this grammar file.  Jison draws its inspiration from Bison
a parser generator for C and C++.  Jison can recognize LALR(1)
LR(0), SLR(1),
and LR(1)
grammars and includes support for generating LL(1) parse tables.

This is a utility function to assist in setting rules.



Usage



rules(/b/, 'RANDOM') # returns [['/b/', 'return "RANDOM";']]
             		                            
var rules = function() {
  var rules = [];
  for (var i = 0; i < arguments.length; i = i + 2) {
    rules.push([
      arguments[i].toString().slice(1, -1),
      'return "'+arguments[i+1]+'";';
    ]);
  };
  return rules;
};

This is a utility function to assist in setting operators.



Usage



operators('l + -'); // returns [['left', '+', '-' ]];
             		                            
var operators = function() {
  var operators = [];
  for (var i = 0; i < arguments.length; i++) {
    var match = /^(l|r)\s*(\S[\s\S]*)/.exec(arguments[i]);
    operators[i] = [match[1] == 'l' ? 'left' : 'right'];
    operators[i] = operators[i].concat(match[2].split(/\s+/));
  };
  return operators;
};

Math constants and functions



The (MDN Documentation)
for javascript is worth looking up because we retain many Javascript
math constants and functions.



In regards to math Slogo differs from Javascript in a number of ways:



    

  1. In Slogo every math constant and function is directly available in
the main namespace.
    


    PI == 3.141592653589793
    


    max 2, 5, 3 == 5
    2. 

  2. Exponents have their own operator, the ^ operator
    


    3 ^ 2 == 9
    3. 

  3. Factorials have their own operator, the ! operator
    


    4! == 4 * 3 * 2 * 1
    4. 

  4. The percent sign is used for percentages.
    


    50% == 0.50
    5. 

  5. The modulus operator is still an operator but it has a longer name
    


    3 mod 2 == 1
    6.
